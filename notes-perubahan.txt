Teori Git & Github (Catatan)
 
1. Git & Github adalah suatu hal yang berbeda 
2. Git Adalah VCS (version control system) Atau source code management 
sistem yang mengelolah perubahan dari sebuah dokumen 
& memungkinkan bekerja
berkolaborasi dengan lebih baik & mengetahui yang melakukan dan kapan
Sebuah perubahan terjadi 
& memungkinkan kembali kekeadaan sebelum perubahan
file di dalam folder (repository / repo) 
& riwayat perubahan file disimpan 
menggunakan rangkaiyan commit

GitHub
1. Layanan cloud untuk menyimpan & mengelola project / repo git.
	  repo : folder project 
	  commit : rekaman dari repository
	  hash : penandaan unik pada sebuah commit 
	  checkout : berpindah kesebuah commit 
	  branch : cabang bebas dari sebuah commit
	  merge : menggabungkan brach
	  remote : sumber yang memiliki repo 
	  clone : mengambil repo dari remote
  	  push : mengirim commit ke repo
	  pull mengambil commit dari repo
